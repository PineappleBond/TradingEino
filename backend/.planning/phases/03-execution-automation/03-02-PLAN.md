  <action>
    Create `internal/agent/tools/okx_attach_sl_tp.go` for attaching stop-loss/take-profit to existing orders:

    1. Define `OkxAttachSlTpTool` struct with `svcCtx *svc.ServiceContext` and `limiter *rate.Limiter`
    2. Implement `NewOkxAttachSlTpTool(svcCtx)` with rate limiter: `rate.NewLimiter(rate.Every(200*time.Millisecond), 1)` (5 req/s)
    3. Implement `Info(ctx)` returning ToolInfo with name `okx-attach-sl-tp` and parameters:
       - `instID` (string, required): 交易对
       - `ordId` (string, required): 已有订单 ID
       - `slTriggerPx` (string, optional): 止损触发价格，和 tpTriggerPx 至少填一个
       - `slOrderPx` (string, optional): 止损委托价格，留空表示市价单
       - `tpTriggerPx` (string, optional): 止盈触发价格，和 slTriggerPx 至少填一个
       - `tpOrderPx` (string, optional): 止盈委托价格，留空表示市价单
    4. Implement `InvokableRun(ctx, argsJSON)`:
       - Call `c.limiter.Wait(ctx)` before API call
       - Parse JSON arguments
       - Validate at least one of slTriggerPx or tpTriggerPx is provided
       - Call `c.svcCtx.OKXClient.Rest.Trade.PlaceAlgoOrder(...)` with `ordType = "sl_tp"`
       - Validate `resp.Code != 0` → return error
       - Validate `algoResult.SCode != 0` → return error (EXEC-06)
       - Format output as Markdown table: algoId, sCode, sMsg
