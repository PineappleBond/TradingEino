  <name>Task 1: Implement okx-place-order-tool</name>
  <files>internal/agent/tools/okx_place_order.go, internal/agent/tools/okx_place_order_test.go</files>
  <behavior>
    - Test 1: Place limit order with valid parameters returns order ID and state
    - Test 2: Place market order (price empty) returns order ID and state
    - Test 3: Invalid parameters (missing required fields) returns error
    - Test 4: OKX sCode != 0 returns error with sCode/sMsg details
  </behavior>
  <action>
    Create `internal/agent/tools/okx_place_order.go` following the established pattern from `okx_candlesticks.go`:

    1. Define `OkxPlaceOrderTool` struct with `svcCtx *svc.ServiceContext` and `limiter *rate.Limiter`
    2. Implement `NewOkxPlaceOrderTool(svcCtx)` with rate limiter: `rate.NewLimiter(rate.Every(200*time.Millisecond), 1)` (5 req/s)
    3. Implement `Info(ctx)` returning ToolInfo with name `okx-place-order` and parameters:
       - `instID` (string, required): 交易对如 ETH-USDT-SWAP
       - `side` (string, required): buy 或 sell
       - `posSide` (string, optional): long/short/net, default net
       - `ordType` (string, required): market/limit/post_only/fok/ioc
       - `size` (string, required): 订单数量（合约张数）
       - `price` (string, optional): 订单价格，limit 和 post_only 必填，market 留空
    4. Implement `InvokableRun(ctx, argsJSON)`:
       - Call `c.limiter.Wait(ctx)` before API call
       - Parse JSON arguments
       - Call `c.svcCtx.OKXClient.Rest.Trade.PlaceOrder(...)`
       - Validate `resp.Code != 0` → return error
       - Validate `order.SCode != 0` → return error (EXEC-06 requirement)
       - Format output as Markdown table with fields: ordId, clOrdId, tag, state, sCode, sMsg
    5. Return `("", err)` for errors, not `(err.Error(), nil)`
  </action>
  <verify>
    <automated>go test ./internal/agent/tools/... -run TestOkxPlaceOrder -v</automated>
  </verify>
